# ts-cli

CLI к системе типов **Essensio** поверх движка
[`@essensio/engine`](https://github.com/essensio/ts-engine) (репо `ts-engine`).
Универсальный интерфейс: на вход — файл-**схема** в нотации
[`essensio/notation`](https://github.com/essensio/notation); CLI находит
управляемую **сущность** (кортеж-тип с `#`) и даёт интерактивный ввод **любых**
кортежей этой шапки в её таблицу (заводится под `#`). Каждое поле и
кортеж целиком проверяются тем же движком (значение допустимо ⟺ проверка не падает).

Языковой префикс `ts-` (как у `ts-engine`) — на случай CLI на других языках;
npm-пакет — `@essensio/cli`.

## Состав

- [`src/cli.ts`](src/cli.ts) — интерактивный CLI: чтение схемы и данных, ввод и проверка кортежей, запись в хранилище.
- [`src/describe.ts`](src/describe.ts) — интерпретатор над AST: выражение-предикат → человекочитаемый текст. Тип остаётся чистым, текст **выводится** из структуры.
- [`src/control-hints.ts`](src/control-hints.ts) — интерпретатор над AST: тип поля → спецификация контрола формы (min/max, options, pattern…). Подсказка для UI; авторитетная проверка — всё равно движок.

`describe` и `control-hints` — чистые подсистемы поверх AST; здесь же, пока не
понадобятся отдельной презентационной библиотекой (напр. веб-конструктору форм).

## Запуск

```bash
npm install                                      # @essensio/engine (из GitHub) + tsx/typescript
npm run tasks                                     # демо: таск-трекер (tasks/schema.tt + tasks/data.tt)
npm run cli -- <схема.tt> --data <файл>           # любая другая схема
#  [a] добавить · [e] изменить · [l] список · [q] выход
npm test                                          # node:test (describe, control-hints)
npm run typecheck                                 # tsc (строгий режим)
```

Хранилище `--data` — в формате литерала-отношения нотации (читается тем же
движком). Демо-схема и данные — в [`tasks/`](tasks/).

## Зависимость от движка

`@essensio/engine` подключён как git-зависимость на `essensio/ts-engine` и
потребляется как TypeScript-исходник (через `tsx`, как и весь проект). При
изменении публичного API движка — обновляются импорты здесь.
